Big O Notation

How the runtime of an algorithm grows as the inputs grow

We say that an algorithm of O(f(n)) if the number of simple operations the computer has to do is eventually less than a constant times f(n), as n increases.

  • f(n) could be linear (f(n) = n)
  • f(n) could be quadratic (f(n) = n2)
  • f(n) could be constant (f(n) = 1)
  • f(n) could be something entirely different!
Big O Notation: O(1)

Always 3 operations

          
            function addUpToo(n) {{ '{' }}
              return n * (n + 1) / 2;
            {{ '}' }}
Big O Notation: O(n)

Number of operations is bounded by a multiple of n (say, 10n)

          
            function addUpTo(n) {{ '{' }}
              let total = 0;
              for (let i = 1; i <= n; i++) {{ '{' }}
                total += i;
              {{ '}' }}
              return total;
            {{ '}' }}
Big O Notation: O(n2)

O(n) operation inside of an O(n) operation.

          
            function printAllPairs(n) {{ '{' }}
              for(let i = 0; i < n; i++) {{ '{' }}
                for(let j = 0; j < n; j++) {{ '{' }}
                return [i, j];
                {{ '}' }}
              {{ '}' }}
            {{ '}' }}
